# 第二章修改意见

本文档仅针对 `/Users/miles/Desktop/大论文/BUPTGraduateThesisLatexTemplate/Chapter/Chapter_02_Fundamentals.tex` 的第二章内容进行检查，不涉及其他章节。

## 一、总体判断

第二章整体框架已经成型，能够为后文的单目标融合定位和多目标定位提供基础铺垫，章节结构也基本完整。但从审校结果看，当前版本仍存在以下三类问题：

1. 个别表述存在概念性风险，可能影响论文论证闭环。
2. 部分术语、公式说明和英文混写不够规范，降低成稿严谨性。
3. 某些基础理论段落写得偏“通用教材式”，与本文研究对象的绑定还可以进一步加强。

其中，最值得优先处理的是 **PDR 在本文场景中的角色定义** 和 **Starlink 信号参数描述的准确性**。

## 二、需要优先修改的问题

### 1. PDR 在“非合作式非法终端定位”场景中的来源不清，存在概念闭环风险

- 位置：第 277 行到第 285 行左右
- 对应小节：`2.3.5 PDR 定位`
- 问题性质：概念一致性问题，优先级高

当前文字直接把 PDR 作为经典定位方法之一引入，并在后文默认其可以作为融合分支参与目标定位。但本文研究对象是“地面非法低轨卫星终端的非合作式无源定位”，在这种任务设定下，监测系统通常无法直接获得目标终端自身的步长、航向或惯导信息。因此，如果这里的 PDR 指的是“目标端 PDR”，则观测来源不成立；如果指的是“无人机平台自身运动递推”或“外部辅助轨迹信息”，则需要明确说明其观测主体和用途。

- 修改建议：
  - 明确说明 PDR ch的信息来源。
  - 如果 PDR 实际是“目标端可获得的外部辅助轨迹信息”或“仿真中可用的辅助分支”，应在第二章就写清楚假设条件，避免后文读者质疑其可获得性。
  - 如果 PDR 实际并非本文必要核心观测量，建议弱化其“经典定位方法”地位，改写为“辅助轨迹约束信息”更稳妥。

### 2. PDR 英文全称写错

- 位置：第 280 行
- 原文：`位置轨道预测（Position Dead Reckoning, PDR）`
- 问题性质：术语错误，优先级高

`PDR` 的常见规范展开不是 `Position Dead Reckoning`。当前写法不标准，会被审稿或答辩老师直接挑出。

- 修改建议：
  - 若沿用定位领域常见写法，建议改为 `Pedestrian Dead Reckoning (PDR)`。
  - 但结合本文场景，如果这里并非“行人惯导定位”，则更建议不要强行保留 `PDR` 这一术语，而应按真实信息源重新命名。

### 3. Starlink 信号参数定义不够严谨，`F_s` 被写成“信道带宽或等效采样率”不准确

- 位置：第 159 行到第 166 行
- 对应公式：式 (2-10)、式 (2-11)
- 问题性质：技术表述不严谨，优先级高

当前写法中：

- `T = N / F_s`
- `F = F_s / N`
- 并说明 `F_s` 表示“信道带宽或等效采样率”

这里把“带宽”和“采样率”混写为同一个量，技术上不够严谨。二者在某些理想化离散基带条件下可以数值相关，但并不应在论文里直接写成同义替换，否则容易被质疑基本概念不清。

- 修改建议：
  - 明确 `F_s` 是采样率，`B` 是带宽，避免混用。
  - 如果这里只是给出离散基带 OFDM 的标准表达式，建议直接写成“在基带离散化建模中，令采样率为 `F_s`，则 ...”。
  - 如果要强调与 Starlink 实际体制的关系，建议再补一句“本节只保留后续信号建模所需的抽象参数，不等同于完整协议复现”。

## 三、明显不规范或不正确的地方

### 5. 中文语境中混入英文句点

- 位置：第 411 行
- 原文：`知识蒸馏建立在教师—学生框架之上.`
- 问题性质：格式问题，优先级中

这是典型的排版清稿残留，虽然不影响理解，但会影响成稿质感。

- 修改建议：
  - 改为中文句号 `。`

### 6. AoA 观测模型使用 `arctan((y-y_i)/(x-x_i))`，存在象限歧义

- 位置：第 223 行到第 230 行
- 对应公式：式 (2-18)
- 问题性质：公式表达可改进，优先级中

当前 AoA 几何模型采用普通反正切形式。该写法在理论说明层面可以勉强使用，但它存在象限歧义，且在 `x = x_i` 附近会带来表达不稳定问题。严格写法通常应使用双参数反正切形式。

- 修改建议：
  - 建议改为 `\theta_i = \mathrm{atan2}(y-y_i, x-x_i) + \epsilon_{\theta,i}`。
  - 如果担心 `atan2` 过于工程化，至少在文字中补一句“实际实现中需结合象限信息确定方位角”。

### 7. 第二节标题是“传播机理”，但内容仍偏少，和标题张力不完全匹配

- 位置：第 74 行到第 172 行
- 对应节标题：`2.2 复杂城市环境下的低轨卫星传播机理`
- 问题性质：结构-内容匹配度问题，优先级中

目前这一节主要包含两块内容：

1. 多径、时延扩展、衰落。
2. Starlink OFDM 基本表达式。

从“传播机理”这个标题看，读者会预期这里至少还应更明确地点出：

- 视距 / 非视距切换机制；
- 城市峡谷遮挡效应；
- 高频段链路在复杂城市场景下对遮挡、反射的敏感性；
- 这些机理分别如何影响 RSSI、AoA、TOA/TDOA 与后续 Radio Map。

当前文本有部分提及，但没有形成更清晰的小层级，因此这一节会显得“标题很大，内容偏窄”。

- 修改建议：
  - 不一定要大改篇幅，但建议在 2.2 节中补一个更清晰的过渡段，明确“传播机理”对后续观测量退化和特征构造的作用链条。
  - 如果时间允许，可考虑把标题改得更贴合现有内容，例如“复杂城市环境下的传播特性与 Starlink 信号基础”。

## 四、可以明显提升论文质量的改进点

### 8. TOA / TDOA 小节建议更明确地点出“对本文场景适用性有限”

- 位置：第 243 行到第 275 行
- 对应小节：`2.3.3 TOA 定位`、`2.3.4 TDOA 定位`
- 问题性质：论证收束不足，优先级中

第二章绪论和后续章节都已经反复强调：本文场景是非法、非合作、复杂城市环境下的无人机定位问题，且 Starlink 类信号还存在时钟稳定性和同步条件上的约束。但 TOA/TDOA 这里目前更像是在做一般教材式介绍，对“为什么它们在本文里不是主路线”交代得还不够强。

- 修改建议：
  - 在 TOA 小节结尾加一句，明确其对绝对同步和时间标定依赖较强，因此在本文研究场景下更适合作为理论参照而非主方法。
  - 在 TDOA 小节结尾进一步指出，尽管它弱化了发射时刻未知问题，但仍高度依赖多站严格同步与稳定传播条件，因此在复杂城市环境和低成本无人机平台下存在工程限制。

### 9. “高维指纹特征”在第二章中被提及，但正文主线已删去独立第三章，建议弱化遗留痕迹

- 位置：第 43 行
- 原文：`以及后续章节中涉及的高维指纹特征等`
- 问题性质：版本残留风险，优先级中

当前 `main.tex` 已不再包含独立的“深度射频指纹单目标定位章节”，因此第二章这里再主动强调“高维指纹特征”，会让读者预期正文中存在对应的大篇幅展开。若后文只保留少量相关内容，这里会显得像旧版结构残留。

- 修改建议：
  - 若全文已确定删除独立指纹章节，建议这里把“高维指纹特征”改为更宽泛的“后续方法所需的特征表示”。
  - 若后文仍保留较多指纹建模内容，则建议在绪论和目录结构中同步说明，不要只在第二章孤立出现。

### 10. 性能评估指标命名可进一步规范

- 位置：第 478 行到第 500 行
- 问题性质：表述规范性问题，优先级低

`Mean Error` 不是错，但比起 `平均定位误差` 或 `Mean Localization Error` 略显随意；另外本节说“第四章多目标定位实验”会使用 CDF，但按当前主文件编排，自动编号上这实际上是正文第四章，不是文件名上的 `Chapter_05`，这一点在全文中最好统一口径。

- 修改建议：
  - 统一改成“平均定位误差”或“Mean Localization Error”。
  - 检查全文章节称呼是否始终按最终排版编号书写，而不是按文件名编号书写。

## 五、建议的修改顺序

建议按以下顺序处理第二章：

1. 先处理 PDR 定义与信息来源问题，这是第二章里最可能引发答辩质疑的点。
2. 再修正 Starlink 信号参数说明中的不严谨表达，避免基础概念被质疑。
3. 之后统一清理英文残留、标点和公式细节问题。
4. 最后再补强 TOA/TDOA 的适用性限制，以及 2.2 节与全文主线之间的衔接表述。

## 六、结论

第二章当前最大的风险不在“排版”，而在 **个别基础概念与本文场景之间还没有完全对齐**。如果只做表面润色，章节会显得完整；但如果要达到终稿或答辩稿质量，建议至少把以下两点彻底处理干净：

1. `PDR` 到底是什么、来自哪里、为什么在非合作非法终端定位场景下可以进入融合框架。
2. `Starlink` 小节中的信号参数和建模假设到底是“通用 OFDM 抽象”还是“与实际体制强绑定的结构描述”。

这两点一旦处理清楚，第二章的理论铺垫会更扎实，也能更好支撑后文方法章节。
