# 第三章修改意见

本文档仅针对当前论文实际第三章，即 [Chapter_04_Fusion.tex](/Users/miles/Desktop/大论文/BUPTGraduateThesisLatexTemplate/Chapter/Chapter_04_Fusion.tex)《基于物理感知深度强化学习的单目标自适应融合定位方法》提出修改建议，不涉及其他章节。

## 一、总体判断

第三章目前已经具备较完整的章节骨架，问题定义、方法结构和实验叙述也基本成型，但仍存在三类比较关键的问题：

1. 存在若干影响方法可信度的概念性问题，尤其是 `PDR` 信息源在“非合作式地面非法终端定位”场景下的可获得性，以及 `PA-DQN` 中“连续动作”与 `DQN` 离散动作框架之间的冲突。
2. 存在若干会直接影响论文严谨性的表达问题，包括训练/推理阶段边界不清、奖励函数依赖真值但未交代监督来源、部分指标定义不充分。
3. 实验与图表部分残留明显的占位式写法、未完成引用和伪代码环境错误，已经接近“清稿前必须处理”的程度。

建议优先修正“概念正确性”和“方法前后一致性”问题，再处理实验呈现与清稿问题。

## 二、重点问题与修改建议

### 1. `PDR` 观测源的可获得性与场景设定存在根本冲突

- 位置：
  - 第 7-14 行
  - 第 28-30 行
  - 第 144-146 行
  - 第 195-205 行
- 问题性质：概念性问题，优先级最高
- 问题说明：
  当前第三章明确将研究对象定义为“复杂城市环境下，利用低成本无人机平台对地面非法低轨卫星终端进行非合作式无源单目标连续定位”，但后文反复把 `PDR` 作为三类核心观测之一，并描述为“利用惯性测量信息表征目标运动过程中的短时连续性”。

  这一设定在现有表述下存在明显逻辑冲突：如果目标是“地面非法终端”且“非合作”，那么论文需要解释无人机平台如何获得该目标的惯性测量数据，否则 `PDR` 这一信息源在问题定义层面就不成立。当前写法容易让评审直接追问：

  - 非法终端并不会主动上报 IMU/计步/航向信息；
  - 无源监管场景下也无法默认获取目标自身的惯性递推轨迹；
  - 若 `PDR` 实际来自无人机自身体位/巡检航迹，则它不应被写成“目标的 PDR 定位结果”。

- 修改建议：
  只能二选一，不能继续维持当前模糊写法。

  方案 A：
  彻底取消 `PDR` 作为目标侧观测源的表述，把第三章改写为 `RSSI + AoA + 运动先验/轨迹预测` 的融合框架。此时应统一把 `PDR` 改为“运动学预测分支”“轨迹递推分支”或“状态先验分支”，并交代该分支来源于上一时刻定位结果、无人机观测几何约束或运动模型，而非目标自带惯导。

  方案 B：
  若作者确实有可解释的数据来源，则必须在第三章开头明确补出前提假设，例如“目标运动先验来自某类外部伴随观测”“PDR 实际来自目标关联载体的可获取惯导信息”等，并清楚说明为什么这仍可归入本文场景。否则不建议继续使用 `PDR` 术语。

  从论文防守角度看，更推荐方案 A。

### 2. `PA-DQN` 的“连续动作空间”与 `DQN` 框架前后冲突

- 位置：
  - 第 521-570 行
  - 第 612-641 行
  - 第 674-676 行
  - 第 711-723 行
  - 第 880 行
- 问题性质：方法论冲突，优先级最高
- 问题说明：
  第 3.4.2 节明确写的是“增量式连续动作空间”，动作定义为连续增量向量
  `[\Delta \omega_t^{RSSI}, \Delta \omega_t^{AoA}]^T`。

  但第 3.4.4 节又明确写 `PA-DQN` 使用 `DQN`，网络输出“各候选动作对应的 Q 值估计”，这对应的是离散动作集合。随后伪代码又使用 `\varepsilon-greedy` 直接选择动作，第 711 行附近实验设置甚至写成“动作采用增量式离散更新方式”，第 722 行表格也写成“增量式离散更新”，而第 880 行章末总结又回到了“增量式连续动作空间”。

  这说明当前文本中至少同时存在三套互相冲突的动作设定：

  - 连续动作；
  - 候选离散动作的 DQN；
  - 增量式离散更新。

  这一点如果不改，会直接削弱整章可信度。

- 修改建议：
  必须统一成一套自洽方案。

  方案 A：
  保留 `DQN`，则动作必须离散化。可改写为“增量式离散动作空间”，例如对每个权重只允许 `{-\Delta, 0, +\Delta}`，形成有限动作集合，再由 `DQN` 输出每个离散动作的 Q 值。这样与 `\varepsilon-greedy`、目标网络、经验回放完全一致，也最容易与现有基线结构衔接。

  方案 B：
  若作者确实想保留“连续动作”，则不能继续称为标准 `DQN`，而应改成适合连续动作的算法，例如 `DDPG`、`TD3`、`SAC`，同时网络结构、损失函数、策略更新公式和伪代码都要重写。这个改动量较大，不建议在终稿阶段临时展开。

  从当前稿件完整性看，更推荐方案 A：把“连续动作”统一改为“细粒度离散增量动作”。

### 3. `RL-IFA` 与 `PA-DQN` 的训练/部署边界需要说清楚

- 位置：
  - 第 237-257 行
  - 第 356-376 行
  - 第 455-470 行
  - 第 575-605 行
  - 第 709-711 行
- 问题性质：论证不完整
- 问题说明：
  第三章中强化学习奖励都依赖真实位置误差，例如：

  - `RL-IFA` 用真实位置构造误差状态和奖励；
  - `PA-DQN` 的精度项也直接使用定位误差 `\varepsilon_t`。

  这在离线仿真训练中没有问题，但如果第三章强调“在线自适应融合定位”，则需要明确区分：

  - 训练阶段：有真值，可计算奖励；
  - 部署阶段：通常无真值，只执行前向决策，不再在线更新。

  当前第 709-711 行写“离线训练、在线推理”，方向是对的，但整章其他位置仍频繁使用“在线更新”“闭环在线更新”等措辞，容易让人误以为部署时也在继续用真值更新策略。

- 修改建议：
  在方法总述和实验设置中增加统一说明，例如：

  “本文采用仿真监督下的离线强化学习训练与部署阶段固定策略推理相结合的方式。训练阶段利用仿真真值构造奖励；部署阶段仅根据当前状态输入执行策略前向推理，不再依赖真实位置进行在线参数更新。”

  同时将“在线更新”“闭环在线更新”等表述统一改成“训练阶段闭环更新”或“策略学习阶段闭环更新”，避免概念外溢。

## 三、建议的修改顺序

建议按下面顺序处理第三章：

1. 先处理 `PDR` 场景设定问题，决定到底保留还是重命名为运动先验分支。

4. 最后补齐实验细节、指标定义和文字清稿。

## 四、简要结论

第三章目前最大的问题不是“写得不够多”，而是“核心方法设定还没有完全自洽”。如果只做语言润色而不先解决 `PDR` 可获得性和 `PA-DQN` 动作空间冲突，这一章在答辩或盲审中都容易被直接抓住关键漏洞。相反，只要先把这两处问题统一清楚，再清掉实验中的占位痕迹，这一章整体质量会明显提升。
