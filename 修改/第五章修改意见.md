# 第五章修改意见

审校对象：`/Users/miles/Desktop/大论文/BUPTGraduateThesisLatexTemplate/Chapter/Chapter_05_Multi_Target.tex`

审校范围：仅限第五章内容，不涉及其他章节。

## 一、总体判断

第五章当前最大的问题不是局部文字，而是**章节主线发生了明显漂移**。

前半章将本章定义为“基于无人机信号成像与跨模态视觉检测的多目标并发定位方法”，技术路线明确指向：

1. 离散 RSSI 观测重构为 Radio Map；
2. 在图像域进行目标检测；
3. 完成多目标并发定位。

但从第 3 节开始，正文主线转为“多无人机随机掉点条件下的教师—学生知识蒸馏协同定位”，并进一步落到了**图神经网络下的单终端栅格分类问题**。这会导致本章在题目、方法、建模、实验和结论五个层面都不完全一致，属于优先级最高的问题。

## 二、需要优先修改的核心问题

### 1. 章节题目、前半章承诺的技术路线，与后半章实际方法不一致

- 位置：
  - 第 1 行：章节标题
  - 第 151-224 行：明确写成“Radio Map + 图像域检测 + Anchor-Free YOLOv7”
  - 第 345 行以后：开始转入“知识蒸馏的鲁棒多无人机协同多目标定位”
  - 第 491-541 行：核心网络实际上是图神经网络 / 图注意力网络
- 问题说明：
  - 第五章标题强调“跨模态视觉检测”，第 222 行也明确写了“改进的 Anchor-Free YOLOv7 检测网络”。
  - 但后文并没有继续展开 YOLOv7 的网络结构、检测头、标签构造、损失函数、输出框到目标坐标的映射关系，也没有给出真正的检测实验。
  - 相反，第 3 节和实验部分完整展开的是“多无人机掉点 + GAT + 知识蒸馏 + 栅格分类”的另一套方法。
- 影响：
  - 读者会认为第五章像是两套不同工作的拼接，主线不闭环。
  - 这一问题会直接削弱章节完整性和论文说服力。
- 修改建议：
  - 二选一，必须统一：
  - 方案 A：保留“跨模态视觉检测/多目标定位”主线。则需要把第 3 节改回 YOLOv7 检测网络设计、标签分配、解耦头、损失函数与检测实验，删去或弱化当前掉点蒸馏内容。
  - 方案 B：保留“多无人机掉点鲁棒协同定位”主线。则应重写章节标题、首节问题定义、第二节结尾和本章小结，把“视觉检测”“多目标并发定位”改成与蒸馏/GAT一致的表述。
  - 不建议维持现状。

### 2. “多目标并发定位”在后半章被改写成“单目标栅格分类”，任务定义前后不一致

- 位置：
  - 第 10-31 行：本章定义为“多个非法终端同时活动”的多目标并发定位
  - 第 391-396 行：`对于任一目标终端，系统最终输出其位于各栅格的概率分布向量`
  - 第 396 行：`最终定位结果由最大概率对应的栅格标签给出`
- 问题说明：
  - 前文研究对象是“目标集合 \(\mathcal{P}\)”和“目标数量 + 各目标位置”。
  - 但后文模型输出是单个 \(M\) 维概率向量，且用 `argmax` 给出唯一栅格标签，本质上是**单目标多分类定位**，不是多目标检测或多目标集合预测。
  - 文中缺少目标数量估计、多目标标签表示、集合匹配、NMS 或多实例输出机制。
- 影响：
  - 这会让“多目标并发定位”这个核心表述在方法层面站不住。
- 修改建议：
  - 如果真实工作是单终端掉点鲁棒定位，应把本章后半的“多目标”表述统一改成“终端定位”或“协同定位”。
  - 如果真实工作确实是多目标任务，则必须补充：
  - 多目标标签表示方式；
  - 多目标输出形式；
  - 多目标损失函数；
  - 目标数估计与位置恢复流程；
  - 多目标评价指标（如 Precision、Recall、mAP、定位偏差）。

### 3. 输入表征前后冲突：前面是 Radio Map / 图像检测，后面变成节点特征图模型

- 位置：

- 问题说明：

  - 这三者不是同一个概念，但文中没有交代它们之间的转换关系。

  - 明确固定一种输入主线，并统一全文术语。
  - 若最终采用 GAT，则应写清：原始观测 -> 节点特征构造 -> 动态图 -> 图模型输出。
  - 若最终采用 Radio Map + 检测网络，则应删除图输入建模相关段落。

## 六、结论

如果只从“局部错字”层面修改，第五章仍然会留下明显的结构性问题。当前最关键的不是修几个词，而是先决定这一章到底要写：

1. **多目标视觉检测定位**，还是
2. **多无人机掉点鲁棒协同定位**。

这两个方向都可以成立，但不能在同一章里以当前这种方式并行展开又互不闭环。建议先统一主线，再做文字级清稿。
