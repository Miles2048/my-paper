## 总体判断

第四章当前**不是简单的文字润色问题**，而是存在比较明显的**章节目标漂移、方法链路不一致、实验闭环不足**等问题。

优先级建议如下：

1. 先解决“本章到底研究什么”的统一性问题。
2. 再解决“输入是什么、模型是什么、输出是什么”的技术链路一致性问题。
3. 最后处理实验补充、术语统一和笔误清理。

如果不先处理前两项，后续局部润色意义有限，因为读者会先在章节逻辑上产生疑问。

## 一、最需要优先修改的核心问题

### 1. 章节标题、问题定义和后文方法不一致

**问题位置：**

- 第 1 行标题
- 第 31 行
- 第 151 行到第 224 行
- 第 345 行以后

**当前表现：**

- 标题写的是“基于无人机信号成像与跨模态视觉检测的多目标并发定位方法”。
- 前半章也一直在铺垫“多目标并发定位”“图像域检测”“区域级判别”“Anchor-Free YOLOv7”。
- 但从第 345 行开始，方法主体突然变成“多无人机随机掉点下的知识蒸馏协同定位”，后续网络实现是图注意力网络 + 蒸馏分类，不再是视觉检测网络，也不再真正对应“多目标并发检测”。

**为什么这是问题：**

- 读者会预期本章主方法是“Radio Map + 目标检测网络”。
- 结果正文主实验却变成“多无人机掉点 + GAT + KD 的栅格分类定位”。
- 这会造成题目、摘要、绪论、章标题和正文主体之间的错位，属于结构性问题。

**修改建议：**

二选一，必须统一：

- 方案 A：保留当前“多无人机掉点鲁棒协同定位”主体。
  - 则应把章标题、开头问题定义、4.1 节铺垫、4.2 节表述全部改成围绕“Radio Map 表征 + 多无人机协同 + 掉点鲁棒蒸馏定位”展开。
  - 同时删除或弱化“跨模态视觉检测”“Anchor-Free YOLOv7”“目标检测区域框”这些没有真正落地的表述。
- 方案 B：保留当前标题“跨模态视觉检测的多目标并发定位方法”。
  - 则必须补出真正的检测网络设计、标签定义、检测损失、输出框/中心点表示、检测指标和检测实验。
  - 目前工作量明显更大。

**建议采用方案 A。**
原因是当前现有正文、实验和结论，已经更接近“多无人机掉点鲁棒定位”而不是“YOLO 检测论文”。

### 2. 前文承诺使用 Anchor-Free YOLOv7，但正文没有实现与验证

**问题位置：**

- 第 222 行

**当前原意：**

- 明确写了“利用改进的 Anchor-Free YOLOv7 检测网络在图像域内识别真实目标响应区域”。

**实际情况：**

- 后文没有 YOLOv7 的网络结构。
- 没有解耦头、标签分配、检测框表示、损失函数。
- 没有 mAP、Precision、Recall、F1 等检测指标。
- 实验全部是定位误差和掉点鲁棒性，没有任何检测实验。

**为什么这是问题：**

- 这是典型的“方法承诺存在，但正文没有兑现”。
- 对答辩老师和评审来说，这是非常容易被问到的点。

**修改建议：**

- 如果采用上面的方案 A，应删除这一句中的 `Anchor-Free YOLOv7`，改为更稳妥的表述，例如：
  - “再结合后续协同定位模型完成目标位置判定”
  - 或“作为后续鲁棒定位模型的统一空间表征输入”
- 同时把“跨模态视觉检测”改成“空间信号成像与鲁棒协同定位”之类更符合现稿的方法名。

### 3. 本章内部“输入表示”前后不一致

**问题位置：**

- 第 151 行到第 224 行
- 第 361 行
- 第 405 行
- 第 491 行到第 541 行
- 第 570 行到第 574 行
- 第 755 行

**当前不一致表现：**

- 一处说输入是 `Radio Map` 图像。
- 一处说教师/学生输入是“完整 Radio Map / 残缺 Radio Map”。
- 但后文真正的模型实现，却是图结构输入，节点特征为 `X_{i,j}=[x_i,y_i,z_i,R_{i,j}]`，核心网络是 GAT。

**为什么这是问题：**

- `Radio Map` 输入和 `图节点特征` 输入是两种完全不同的数据表示。
- 如果模型实际输入是图节点特征，那么前文大量关于“图像域检测”的叙述就需要收缩。
- 如果模型实际输入是 Radio Map 图像，那么 GAT 架构又缺少中间转换说明。

**修改建议：**

必须明确一条唯一的数据链路，建议写成：

1. 无人机空间扫描得到离散 RSSI 样本。
2. 为了进行区域表征与场景可视化，先构建 Radio Map。
3. 在掉点鲁棒协同定位任务中，真正送入 GAT 的是节点级观测特征，而非整张图像。
4. Radio Map 的作用是场景表征、数据生成与物理解释；GAT 的作用是残缺节点条件下的定位决策。

如果想保留 Radio Map 与 GAT 的联系，建议补一句：

- “本章的 Radio Map 主要用于传播场重构、场景可视化和统一表征说明；实际协同定位网络输入为从多无人机观测中抽取的节点级特征图结构。”

否则当前读者很容易理解成“图像进网络”，但后文代码思路其实是“图进网络”。

## 二、技术链路上可以改进的地方

### 5. Radio Map 重构部分缺少关键数学细节

**问题位置：**

- 第 176 行
- 第 317 行到第 325 行
- 第 753 行

**当前问题：**

- 文中多次提到“空间映射、插值、平滑”“高斯平滑连续化表达”。
- 但没有给出明确的高斯核公式、核宽度、插值方式、空值补全规则、归一化方式。

**为什么这是问题：**

- 本章前半部分的创新基础就是“离散观测到 Radio Map”。
- 如果这部分只有概念描述，没有明确计算方式，方法会显得偏虚。

**修改建议：**

- 建议补一个小段公式，至少说明：
  - 网格赋值规则；
  - 对空栅格的处理方式；
  - 高斯平滑公式；
  - 核参数 \(\sigma\) 或窗口大小。

例如可以补成：

```tex
\tilde{S}(u,v)=\sum_{m}\sum_{n} S(m,n)\,G_{\sigma}(u-m,v-n)
```

并解释 `G_\sigma` 的定义、归一化和参数选择依据。

## 七、结论

第四章当前最大的问题不是“写得不够多”，而是“写了两套尚未完全统一的方法叙事”：

- 前半章是 `多目标并发 + Radio Map + 视觉检测`
- 后半章是 `多无人机掉点 + GAT + 知识蒸馏`

这两条线分别都可以成立，但现在被放在同一章里时，缺少足够强的过渡与统一定义，导致整章显得“铺垫一种方法，实验验证另一种方法”。

建议优先把整章收敛为一条主线，再做局部润色。只要主线统一，这一章的可读性和答辩说服力都会明显提升。
